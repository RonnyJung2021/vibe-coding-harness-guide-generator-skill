## 一切的起点

当你第一次在 IDE 中打开 Agent 对话框，兴奋地敲下：

> "帮我做一个待办事项应用"

你期待着奇迹发生。但现实往往是：

- Agent 不理解你想要什么
- 任务执行到一半报错，你不知道发生了什么
- 上下文爆炸，你无法控制它的行为
- 你不敢让它执行危险命令，却又不知道如何安全地授权

**这不是你的问题，也不是模型的问题。**

这是 **Harness 的缺失**。

### 什么是 Harness？

**Harness 是 AI Agent 的环境系统** — 它不是提示词，不是工具列表，而是一套完整的工程方法论，让 Agent 在生产环境中：

- **可控制** — 任务边界清晰，操作可拦截
- **可预测** — 状态可落盘，过程可回溯
- **可验证** — 每步有标准，交付有证据
- **可扩展** — 熵增可管理，系统可持续运行

## 核心公式

| 原则 | 说明 |
|------|------|
| **约束 > 智能** | Agent 必须在边界内运行，而非依赖模型的"自觉" |
| **反馈 > 输出** | Agent 理解失败原因，而非仅知道失败发生 |
| **稳定 > 灵活** | 系统可维护、可恢复，比一次性成功更重要 |

> **环境是可复用资产。模型可以替换，Harness 不可跳过。**

## 核心理念

> **不依赖模型临场发挥，而依赖环境与可复跑流程。**

当你有一套完善的 Harness 系统：

- 同一个模型，可以稳定地产出高质量代码
- 同一个任务，可以被不同的 Agent 复现
- 同一个错误，可以被系统地排查和修复

## 快速开始

### 1. 触发指南生成

在 Cursor / Trae 的 Agent 对话框中，描述你的任务：

```
我想做一个用户登录功能，使用 JWT 认证
```

### 2. 获得结构化指南

系统输出分阶段操作步骤，每步包含可直接粘贴的 Agent 指令与验收标准。

### 3. 逐阶段执行

- 复制指令 → 粘贴到 Agent → 批准运行 → 验证结果
- 失败？系统会告诉你具体原因和修复方向

## 与传统方式的对比

| 维度 | 传统方式 | Vibe + Harness |
|------|---------|----------------|
| **任务表达** | 模糊、依赖模型猜测 | 契约明确、边界清晰 |
| **执行过程** | 黑盒、不可回溯 | 状态落盘、每步可验证 |
| **失败处理** | 随机重试、靠运气 | 错误分类、针对性修复 |
| **交付标准** | 依赖人工检查 | 自动验收、证据留存 |
| **可复制性** | 低，依赖个人经验 | 高，流程标准化 |
