## 🎯 问题的起点

当你在 IDE Agent 对话框敲下：

> "帮我做一个待办事项应用"

现实往往是：

- ❌ Agent 不理解你想要什么
- ❌ 执行到一半报错，不知道发生了什么
- ❌ 上下文爆炸，无法控制
- ❌ 不敢授权危险操作

**这不是模型的问题 — 这是 Harness 的缺失。**

### 🔥 Harness 是什么？

**AI Agent 的环境系统** — 一套让 Agent 在生产环境中：
- 🛡️ **可控制** — 任务边界清晰，操作可拦截
- 📊 **可预测** — 状态可落盘，过程可回溯
- ✅ **可验证** — 每步有标准，交付有证据
- 🔄 **可扩展** — 熵增可管理，系统可持续

## ⚡ 核心原则

| 原则 | 说明 |
|------|------|
| **约束 > 智能** | Agent 在边界内运行，而非依赖模型"自觉" |
| **反馈 > 输出** | 理解失败原因，而非仅知道失败 |
| **稳定 > 灵活** | 可维护、可恢复 > 一次性成功 |

> **环境是可复用资产。模型可替换，Harness 不可跳过。**

## ⚡ 快速开始

**1.** 在 Cursor/Trae Agent 对话框描述任务

```
我想做一个用户登录功能，使用 JWT 认证
```

**2.** 获得结构化指南 → 直接粘贴到 Agent 执行

**3.** 每步验证，失败？系统告诉你原因和修复方向
